# MRVP Method-to-Code Mapping

This repository implements **MRVP: Motion-Reset Viability Programming**.

| Paper component | Code path | Notes |
|---|---|---|
| Reset-problem row schema | `mrvp/data/schema.py`, `mrvp/data/dataset.py` | Preferred fields are `reset_state`, `reset_time`, `degradation`, `recovery_world`, `reset_slots`; legacy `x_plus/event_time/deg/world_plus/event_tokens` are aliases only. |
| Reset boundary target | `mrvp/data/reset_targets.py`, `mrvp/data/metadrive_rollout.py`, `mrvp/data/synthetic.py` | Lightweight construction chooses the hardest prefix point by recovery difficulty when full teacher-per-step rollout is unavailable. |
| CMRT | `mrvp/models/cmrt.py`, `mrvp/models/reset_memory.py` | Root-scene memory, action-intervention memory, counterfactual reset slots, reset state mixture and recovery-world/degradation decoders. |
| RPFN | `mrvp/models/rpfn.py`, `mrvp/models/recovery_programs.py` | Program queries attend to reset slots and synthesize closed-loop recovery programs. |
| Degraded rollout | `mrvp/dynamics/degraded_bicycle.py` | `program_rollouts` are generated by differentiable degraded dynamics, not by a direct trajectory head. |
| Funnel certificates | `mrvp/safety/margins.py`, `mrvp/safety/certificates.py` | Road, secondary-collision, stability, control and goal margins are aggregated with funnel values into larger-is-better certificates. |
| Tail-consistent selection | `mrvp/action_selection.py` | For each action, CMRT samples reset problems, RPFN scores programs, and selection maximizes lower-tail CVaR of `max_l certificate`. |
| Evaluation | `scripts/evaluate_predicted_mrvp.py` | Reports `Predicted_MRVP_CMRT_RPFN_LCVaR` and supporting baselines. |
| Legacy compatibility | `mrvp/models/msrt.py`, `mrvp/models/rpn.py`, `mrvp/selection.py` | Deprecated shims for loading old checkpoints or scripts; not recommended in the main path. |

## Main formula implemented

```text
candidate prefix a
  -> CMRT samples R_a = (r_a, z_a, W_a, d_a, Sigma_a)
  -> RPFN builds programs P_a^ell and degraded rollouts
  -> c_a^{m,ell} = min certificate margin
  -> S(a) = LCVaR_beta({ max_ell c_a^{m,ell} }_m)
  -> select argmax S(a) within the harm-comparable candidate set
```
